# 深浅拷贝

# 什么是深浅拷贝？为什么要深拷贝？

说到深浅拷贝就涉及到javascript的数据类型，javascript数据类型分为基本数据类型和引用数据类型两种，

基本数据类型（number,string,undefined,boolean，null,symbol）六种 基本数据类型的值存在栈内存中，

引用数据类型 （array,object,function） 引用数据类型的值存在堆内存中，

栈内存中存放基本类型的变量和引用类型指针的变量 js中的浅拷贝和深拷贝就是对引用数据类型的复制操作，由于引用数据类型的值存在堆内存中， 因此在复制操作中就出现两种情况， 复制引用（栈内存中的引用指针）和复制一个新的引用数据类型（对象）

基本类型

let num = 10;
let num1 = num;
num1 = 11;
console.log(num,num1) // 10 11

上边的代码中 num1是num的副本，它们在内存中占有不同位置但相等的内存控件 所以改变num1为11之后，num还是不会变化

引用类型

let obj = {
  name : "yonghua",
  age:25,
}
let obj1 = obj
obj1.name = "sunyonghua"
console.log(obj.name,obj1.name) // sunyonghua sunyonghua

obj 这个变量存放的其实不是对象的属性和值，而是存放的指向这个对象的引用指针， 所有obj1 = obj这个赋值操作只是把obj对象的指针复制了一份给obj1,因此两个变量指向的都是同一个对象 所以在改变obj1对象的属性值的时候，obj也会被改变

两者区别 浅拷贝只是复制指向对象的指针地址，而不复制对象本身，新旧对象是同一个，共享同一块内存，一个被改变，另一个也会跟着变化 深拷贝相当于创建了一个新的对象，不共享内存，修改新对象，就对象保持不变

浅拷贝

共享内存,复制对象的引用指针

let obj = {
  name : "yonghua",
  age:25,
}
let obj1 = obj
obj1.name = "sunyonghua"
console.log(obj.name,obj1.name) // sunyonghua sunyonghua

深拷贝

复制并创建一个新的对象，不共享内存。

递归复制 方法内传入要拷贝的对象，遍历对象并赋值给新创建的对象，最后返回新对象， 注意这里如果属性值为引用类型就需要递归赋值，不然复制的属性还是指向原对象的引用指针上

let obj = {name: 'yonghua', age: 18, pet: {name: 'huahua'}}

let obj1 = deepClone(obj)

function deepClone(target) {
  // 创建新对象
  let newObj = {}
  // 遍历要拷贝的对象并把每一个属性跟属性值赋值新对象
  for (let i in target) {
    // 判断是否是自身的属性，防止获取到原型链上的属性
    if (target.hasOwnProperty(i)) {
      // 判断属性是否是引用数据类型，如果是引用数据类型，则递归处理
      if (typeof target[i] === 'object') {
        newObj[i] = deepClone(target[i])
      } else {
        newObj[i] = target[i]
      }
    }
  }
  return newObj
}

obj1.name = "sunyonghua"
obj1.pet.name = 'wangwang'

console.log(obj, obj1)

使用JSON的parse和stringify方法

let obj = {name: 'yonghua', age: 18, pet: {name: 'huahua'}}
let newObj = JSON.parse(JSON.stringify(obj))
newObj.name = "sunyonghua"
newObj.pet.name = 'wangwang'
console.log(obj, newObj)

打印结果同上，利用JSON先把对象转为字符串再转成对象, 虽然简单粗暴，但是这个方法有很多弊端，谨慎使用，对function,undefined,RegExp,Error,Date等不友好
