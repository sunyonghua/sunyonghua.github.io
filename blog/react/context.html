    # react Context

    在典型的react应用中，数据都需要props来自上而下的传递，当有多层嵌套时，数据维护起来较为复杂，Context提供了一种在组件内共享值的方式，可以不必显式的通过props传递

    import React, { createContext, useContext } from 'react'

const ThemeContext = createContext('light')

const Context = () => {
  return (
    <>
      Context组件
      <ThemeContext.Provider value="dark">
        <Button/>
      </ThemeContext.Provider>
    </>
  )
}

const Button = () => {
  const themeContext = useContext(ThemeContext)
  console.log(themeContext) // dark
  return (
    <>
      <Icon/>Button<Icon2/>
    </>
  )
}


const Icon = () => {
  const MyContext = useContext(ThemeContext)
  console.log(MyContext) // dark
  return (
    <>
      Icon
    </>
  )
}

class Icon2 extends React.Component {

  // themeContext = useContext(ThemeContext)
  // class 组件用以下方式获取Context,不可使用useContext且此处必须声明静态变量contextType,否则this.context为空对象
  static contextType = ThemeContext;

  render() {
    // 访问this.context
    console.log('icon2' + this.context)
    return (
      <>
        icon2
      </>
    )
  }
}

export default Context

    # displayName

    context 对象接受一个名为 displayName 的 property，类型为字符串。React DevTools 使用该字符串来确定 context 要显示的内容。

    const MyContext = React.createContext(/* some value */);
MyContext.displayName = 'MyDisplayName';

<MyContext.Provider> // "MyDisplayName.Provider" 在 DevTools 中
<MyContext.Consumer> // "MyDisplayName.Consumer" 在 DevTools 中
