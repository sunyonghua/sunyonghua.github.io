# vue 页面上传到服务器页面空白问题

    • vue 使用 h5-history 模式 上传到服务器页面空白什么都看不到找不到问题，后发现去掉 history 之后好了，
    # nginx 配置
location / {
  root   html;
  index  index.html index.htm;
}
    • webpack 打包错误，项目中引入外部 js 使用的 es6 语法，webpack 在打包时并未转换为 es5 语法，ios9 以上是可以打开页面的，ios9 以下是不可以的，最后真机联调定位到问题在于 ios9 以下 js 报错，
    {
    test: /\.js$/,
    loader: 'babel-loader',
    include: [resolve('src'), resolve('test')]
},

    加入如下代码搞定

    {
      test: /\.js$/,
      loader: 'babel-loader',
      // 加入的resolve路径是错误代码的路径
      include: [resolve('src'), resolve('test'),resolve('static'),resolve('/node_modules/vue-preview/src')]
},

# 真机联调方案


    Android

    只需要四个步骤：

    1、先保证 是否安装了chrome浏览器

    2、其次 ，保证 chrome 是否 已经翻墙成功！

    3、最后 使用Android usb真机连接电脑，打开手机usb调试模式

    4、最后的最后，打开chrome ,输入：chrome://inspect

    即可，另外，chrome只支持 android 4.4以上手机！

    IOS

    只需要两步骤：

    1、打开手机端web检查器，设置 -> Safari -> 高级（最底部） -> Web 检查器 打开

    2、确保iOS系统手机接入 MAC 电脑（必须是Mac，window调试不了ios系统）

    3、找到Mac电脑的 Safari浏览器，找到开发者，就会显示的手机的设备

    4、此时此刻就可以使用了，调试各个App上网页版的页面了。
